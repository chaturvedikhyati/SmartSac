🌱 SmartSac Frontend - Quick Start

Choose Your Server

Pick one of the following methods to serve the frontend:

Option 1: Live Server (VS Code) - Recommended for Development

  1. Install the "Live Server" extension in VS Code
  2. Right-click index.html → "Open with Live Server"
  3. Browser will open at http://localhost:5500
  4. Auto-refreshes on file changes

Option 2: Python (Built-in)

cd path/to/smartsac-frontend
python -m http.server 8000

Visit: http://localhost:8000

Option 3: Node.js

npm install -g http-server
http-server .

Visit: http://localhost:8080

Option 4: PHP (Built-in)

php -S localhost:8000

Visit: http://localhost:8000

Option 5: Node.js Express

npm install express
node -e "require('express')().use(require('express').static('.')).listen(3000, () => console.log('Server at http://localhost:3000'))"

Configuration

Important: Update the API endpoint in js/core/api.js

const API_CONFIG = {
  baseURL: 'http://localhost:5000/api',  // Change this!
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
  },
};

Make sure your backend is running on the same URL!

Verify Setup

  1. ✓ Frontend loads at your server URL
  2. ✓ No console errors (F12 → Console)
  3. ✓ Backend API is running
  4. ✓ API baseURL is correctly configured
  5. ✓ Dark mode toggle works (click moon icon)

Production Deployment

Vercel (Easiest)

  1. Push code to GitHub
  2. Connect GitHub to Vercel
  3. Vercel auto-deploys on every push
  4. Update API endpoint in backend settings

Traditional Server

  1. Upload all files to web server via FTP/SFTP
  2. Configure .htaccess or nginx for routing (if using SPA)
  3. Update API endpoint to production backend
  4. Enable HTTPS

Troubleshooting

  • API Error 404: Check backend URL in js/core/api.js
  • CORS Error: Backend needs CORS headers enabled
  • Files not loading: Clear cache (Cmd/Ctrl + Shift + R)
  • Dark mode not working: Check theme.js is loaded
